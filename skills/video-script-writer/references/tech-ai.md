## 🧠 Jargon → Plain English Translation
| Technical Term | Script-Friendly Version |
|---|---|
| Token | A piece of text — roughly a word or syllable |
| Context window | The AI's working memory — how much it can "see" at once |
| RAG | Giving the AI a custom knowledge base to look up answers from |
| Fine-tuning | Training an AI on YOUR specific data so it learns your style |
| Inference | The moment the AI actually runs and generates a response |
| Hallucination | When AI confidently makes something up |
| Embedding | Converting text into numbers so AI can compare meaning |
| Agent | An AI that can take actions, not just answer questions |
| Prompt engineering | The art of talking to AI in a way it understands best |
| Multimodal | AI that can handle text, images, audio, and video together |
| Quantization | Making AI models smaller so they run on your laptop |
| Vector database | A special database that finds things by meaning, not keywords |

## 📐 Structure for Different Tech Videos

### Tutorial / How-To
1. Hook: Show the finished result first ("Here's what we're building")
2. Why it matters: Real-world use case in 30 seconds
3. Prerequisites: Fast list — don't linger
4. Build it: Step-by-step, screen record everything
5. Gotchas: 2–3 common mistakes to avoid
6. Result + extensions: What's possible from here
7. CTA: Link to repo / follow for part 2

### AI Tool Review
1. Hook: Most surprising capability first
2. What is it: 60-second overview
3. Testing it: Live demo, push its limits
4. Where it shines: Best use cases
5. Where it fails: Be honest — this builds trust
6. Who it's for: Specific recommendation
7. Verdict + CTA

### Explainer (Concept)
1. Hook: "Most people think X works like Y — they're wrong"
2. The wrong mental model: What people assume
3. How it actually works: Analogy first, then mechanics
4. Why it matters: Real consequence of understanding this
5. What to do with this knowledge: Practical takeaway
6. CTA

## 🎨 Tone Notes
- Talk to viewers like they're smart but new — respect their intelligence
- Enthusiasm is appropriate — this stuff IS exciting, show it
- Be honest about limitations — it builds authority
- Avoid corporate speak ("leverage synergies") — this kills engagement instantly
- Name drop specific models, tools, version numbers — specificity = credibility
